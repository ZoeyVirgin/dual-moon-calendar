```
### 当前代码架构关键约定

1. **所有组件直接读取 Zustand Store，不通过 props 层层传递**（除少数通用组件如 GridCell 外）
2. **数据流**: Engine(getCalendarDate) → Store(selectedAbs) → CalendarGrid.generateMonthDates → GridCell
3. **事件流**: EventList(状态管理) → EventModal(表单) → useEventStore.addEvent → LocalStorage
4. **CSS 变量体系**: 在 `theme.css` 中用 `:root {}` 定义 Design Token，各组件通过 `var(--xxx)` 引用
5. **类名合并**: `cn()` 工具函数（`src/utils/cn.ts`）封装 clsx + tailwind-merge
6. **TypeScript 类型**: 全局类型在 `src/types/calendar.ts` 和 `src/types/events.ts`
7. **引擎统一入口**: `src/engine/index.ts` 导出所有公开 API，外部组件不直接 import 内部模块
8. **路径别名**: `@/` 映射到 `src/`（在 tsconfig.json 和 vite.config.ts 中配置）
```

   ```bash

   git init
   git add .
   git commit -m "Initial commit: dual-moon calendar V1.0"
   ```
