# 解决方案 — Vercel Serverless Functions 404

## 根因

`api/*/route.ts` + `export function GET()` 是 **Next.js App Router 专用约定**。Vite 项目的 "Other" 预设不会触发 Next.js builder，这些文件被完全忽略——不识别为 Serverless Functions。

## 修复内容

### 1. 文件结构：route.ts → 扁平 .ts

| 旧路径 | 新路径 |
|:--------|:-------|
| `api/ping/route.ts` | `api/ping.ts` |
| `api/auth/route.ts` | `api/auth.ts` |
| `api/events/route.ts` | `api/events.ts` |
| `api/events/[id]/route.ts` | `api/events/[id].ts` |

### 2. 导出格式：Web API → @vercel/node

```typescript
// 旧（Next.js App Router 专用，Vite 下不工作）
export function GET() {
  return Response.json({ ok: true })
}

// 新（Vercel 标准 Serverless Functions）
import type { VercelRequest, VercelResponse } from '@vercel/node'

export default function handler(req: VercelRequest, res: VercelResponse) {
  return res.status(200).json({ ok: true })
}
```

关键变化：
- `export function GET/POST/PUT/DELETE` → `export default function` + `req.method` 分发
- `Request` / `Response.json()` → `VercelRequest` / `res.status().json()`
- `await request.json()` → `req.body`（@vercel/node 自动解析）
- `request.headers.get('Authorization')` → `req.headers.authorization`

### 3. vercel.json：显式 builds + routes

```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "builds": [
    { "src": "api/**/*.ts", "use": "@vercel/node" },
    { "src": "dist/**", "use": "@vercel/static" }
  ],
  "routes": [
    { "src": "/api/(.*)", "dest": "/api/$1" },
    { "handle": "filesystem" },
    { "src": "/(.*)", "dest": "/index.html" }
  ]
}
```

- `builds` 强制 Vercel 用 `@vercel/node` 编译 api/ 下的 TypeScript 文件
- `routes` 确保 `/api/*` 请求路由到 Serverless Functions，其余回退到 SPA 的 index.html

## 前端无需改动

前端 fetch 路径不变（`/api/auth`、`/api/events`、`/api/events/${id}`），与新文件结构完全匹配。

## 部署步骤

### Step 1: 提交代码

```bash
git add api/ vercel.json
git commit -m "fix(api): 转换为 Vercel 标准 Serverless Functions 格式"
git push origin main
```

### Step 2: Vercel Dashboard 确认设置

1. 打开项目 → **Settings** 标签
2. **General** → **Build & Development Settings**：
   - Framework Preset: **Other**（保持不变）
   - Build Command: `npm run build`（vercel.json 已覆盖，此处可留空）
   - Output Directory: `dist`（同上）
3. **Environment Variables** 确认存在：
   - `AUTHOR_PASSWORD` = 你的口令
   - `POSTGRES_URL` 或 `DATABASE_URL` = Neon 连接字符串（@vercel/postgres 自动读取）

### Step 3: 重新部署

1. **Deployments** 标签 → 最近一次部署右侧 **⋯** 菜单 → **Redeploy**
2. 勾选 **Use existing Build Cache** 可不勾（首次建议不勾）
3. 点击 **Redeploy**

### Step 4: 验证

部署完成后访问：

| URL | 期望结果 |
|:----|:---------|
| `https://<域名>/api/ping` | `{"ok":true,"time":...}` |
| `https://<域名>/api/auth` (POST, body: `{"password":"xxx"}`) | `{"success":true,"token":"..."}` |
| `https://<域名>/api/events` | `[]` 或事件数组 |

## 技术说明

- `@vercel/node` 是 Vercel 平台内置 builder，不需要安装到 devDependencies
- `@vercel/node` 使用 esbuild 编译 TypeScript，运行时为 Node.js 20.x
- `@vercel/postgres` 在运行时从 node_modules 解析，不需要打包
- `Buffer.from()` 在 Node.js 运行时正常工作
- 动态路由参数通过 `req.query.id` 获取（如 `api/events/[id].ts`）
